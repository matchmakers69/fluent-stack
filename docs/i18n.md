# Internationalisation — next-intl

This project uses next-intl for two languages.
Default locale: pl (Polish) — no URL prefix.
Second locale: en (English) — /en prefix.
Adding a new locale: add it to routing.locales in
src/i18n/routing.ts and create messages/{locale}.json.

## File locations
- src/i18n/routing.ts    — locale config and locales list
- src/i18n/request.ts    — server-side message loading
- src/i18n/navigation.ts — typed Link, useRouter, usePathname
- messages/pl.json       — Polish strings
- messages/en.json       — English strings

## Rules
- NEVER hardcode user-visible text in components or pages.
  All strings must come from messages/*.json via useTranslations.
- Import Link, useRouter, usePathname ONLY from
  @/i18n/navigation — never from next/link or next/navigation.
- When adding a new page or string, add translations to BOTH
  pl.json and en.json at the same time. Never leave one missing.
- Keep translation keys in English, nested by feature:
  navigation.home, home.title, booking.title etc.
- The [locale] layout handles NextIntlClientProvider.
  Do not add it anywhere else.
- LanguageSwitcher handles all locale switching — never
  implement locale switching logic elsewhere.

## Adding a new translation
1. Add key to messages/pl.json
2. Add same key to messages/en.json
3. Use: const t = useTranslations("namespace")
        t("key")

## Adding a new locale
1. Add locale string to routing.locales in src/i18n/routing.ts
2. Create messages/{locale}.json with all existing keys
3. Add flag emoji to LOCALE_FLAGS in LanguageSwitcher.tsx
4. No other changes needed
